应用

应用类动作用于操作和管理Android应用，包括打开应用、获取当前应用等。

动作列表​

动作类型功能说明
打开应用打开选定的应用
打开链接打开Uri Scheme链接
打开快捷方式打开应用的快捷方式
获取当前应用获取当前运行的应用与界面
是否在应用内判断待测应用或界面是否被包含
文本转应用或界面将文本转为应用或界面值
等待在应用内等待当前处于某个应用或界面内

详细说明​

打开应用​

启动指定的Android应用。

使用场景：自动打开特定应用开始操作。

针脚说明：官方参数信息

  • 应用或界面：要打开的应用或界面，当有写入安全设置权限时，可打开未导出的界面。
  • 动作(Action)：执行动作。
  • 启动标志(Flags)：启动标志。
  • 分类(Category)：动作分类。
  • Uri数据：Uri数据。
  • 参数：参数。
  • 额外信息(Bundle)：额外信息。

打开链接​

打开指定的URI Scheme链接。

使用场景：打开应用的特定页面、打开网页等。

针脚说明：

  • 链接：要打开的链接地址。

打开快捷方式​

打开应用的系统快捷方式。

使用场景：快速启动应用的特定功能。

针脚说明：

  • 应用或界面：应用的内置快捷方式。

获取当前应用​

获取当前前台运行的应用包名和界面名称。

使用场景：判断当前处于 哪个应用。

针脚说明：

  • 应用或界面：输出当前运行的应用界面。
  • 应用名称：输出当前运行应用的名字。

是否在应用内​

判断当前是否在指定的应用或界面内。

使用场景：条件判断、状态检查等。

针脚说明：

  • 应用或界面：要判断的应用列表。
  • 待测应用：要判断的应用或界面，通常连接“获取当前应用”。
  • 结果：布尔值输出，真表示在指定应用/界面内，假表示不在。

文本转应用或界面​

将文本格式的应用信息转换为应用或界面类型的值。

使用场景：动态应用处理。

针脚说明：

  • 包名：应用的包名。
  • 界面类名：应用的界面类名。
  • 应用或界面：应用或界面类型的值。

等待在应用内​

等待直到当前处于指定的应用或界面内。

使用场景：等待应用启动完成、等待页面跳转完成等。

针脚说明：

  • 超时：等待超时时间（毫秒）。
  • 检查间隔：检查条件成立间隔时间（毫秒）。
  • 应用或界面：要等待的应用或界面。
  • 执行：等待达成的分支。
  • 未达成：等待超时时执行的分支。
