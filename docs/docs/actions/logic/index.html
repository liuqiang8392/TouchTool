逻辑

逻辑类动作用于控制任务的执行流程，实现条件判断、循环、分支等逻辑结构。

动作列表​

动作类型功能说明
如果…就…满足条件时执行，否则执行未达成节点
等待条件成立在一定时间内满足条件就执行，否则执行未达成节点
选择执行根据文本值执行对应针脚
手动选择执行弹出选择窗口供使用者手动选择
循环执行…次循环多次执行节点
条件循环执行满足条件时执行，否则执行完成节点
顺序执行执行针脚从左到右顺序执行
随机执行从所有执行针脚中随机一个执行
并行执行多条执行并行执行，直到超时或者达成指定完成数量

详细说明​

如果…就…​

基础的条件判断节点，当条件满足时执行"执行"分支，否则执行"未达成"分支。

使用场景：根据不同条件执行不同操作。

针脚说明：

  • 条件：判断条件。
  • 执行(底部)：条件为真 时执行的分支。
  • 未达成：条件为假时执行的分支。

等待条件成立​

在指定时间内等待条件成立，如果条件成立则继续“执行”，否则执行“未达成”分支。

使用场景：等待某个界面出现、等待某个控件出现等。

针脚说明：

  • 超时：等待超时时间（毫秒）。
  • 检查间隔：检查条件成立间隔时间（毫秒）。
  • 条件：等待的条件。
  • 执行(底部)：条件成立时执行的分支。
  • 未达成：超时后执行的分支。

选择执行​

类似于switch语句，根据输入的文本值选择执行对应的分支。

使用场景：多条件分支处理。

针脚说明：

  • 文本：选择条件值。
  • 默认：没有匹配到任何选项时执行的分支。
  • 添加执行：添加一个执行分支，每个选项对应一个执行分支。

手动选择执行​

弹出选择窗口，让用户手动选择要执行的分支。

使用场景：需要用户参与决策的场景。

记忆位置手动选择

针脚说明：

  • 超时：选择超时时间（毫秒），超时后执行"默认"分支，0为无限等待。
  • 位置：选择窗口显示位置。
    • 记忆位置：显示在记忆位置的窗口，用户选择后记住位置。
    • 手动选择：设置自定义位置。
      • 窗相对屏位置：指定悬浮窗锚点的相对屏幕锚点的位置。
      • 悬浮窗锚点：指定悬浮窗锚点。
      • 屏幕锚点：指定屏幕锚点。
  • 添加执行：添加一个执行分支，每个选项对应一个执行分支。
  • 默认：没有选择任何选项时执行 的分支。

循环执行…次​

指定循环次数，重复执行某个操作。

使用场景：批量操作、重复点击等。

针脚说明：

  • 跳出循环：执行后在下一轮结束循环。
  • 开始值：指定循环开始值。
  • 结束值：指定循环结束值，结束值可以小于开始值，这时为倒序循环。
  • 步长：循环步长，正数。
  • 当前值：当前循环值。
  • 执行(底部)：连接要循环执行的逻辑。
  • 完成：循环完成后执行的分支。

条件循环执行​

当条件满足时持续循环执行，条件不满足时退出循环。

使用场景：持续滚动直到找到目标、持续刷新直到数据更新等。

针脚说明：

  • 条件：循环条件，条件为真时继续循环，否则退出循环。
  • 执行(底部)：连接要循环执行的逻辑。
  • 完成：循环完成后执行的分支。

顺序执行​

将多个执行分支按从左到右的顺序依次执行。

使用场景：需要依次执行多个步骤的场景。

针脚说明：

  • 添加执行：添加一个执行分支。
  • 完成：顺序执行完成后执行的分支。

随机执行​

从多个执行分支中随机选择一个执行。

使用场景：随机操作、模拟人类行为等。

针脚说明：

  • 执行次数：随机执行分支数量。
  • 允许重复：是否允许执行已执行过的分支，开启后可能导致执行次数少于指定次数。
  • 添加执行：添加一个执行分支。
  • 完成：随机执行完成后执行的分支。

并行执行​

同时执行多个分支，可以设置超时时间和完成数量条件。

使用场景：同时监控多个条件、并行处理多个任务等。

针脚说明：

  • 超时：并行执行的超时时间（毫秒），超时后结束所有分支并执行"完成"分支。
  • 完成数：需要完成的执行数量，达到此数量后停止其他执行。
  • 添加执行：添加一个执行分支。
  • 完成：并行执行完成后执行的分支。
